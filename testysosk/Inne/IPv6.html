IPV6

Wymagania instalacyjne systemów Windows zalecane

Testowanie konfiguracji protokołu IPv6 przy użyciu polecenia ping
1. Aby uzyskać konfigurację protokołu IPv6 komputera, otwórz okno wiersza polecenia, a następnie wpisz polecenie netsh interface ipv6 show interface.
2. W wierszu polecenia wpisz polecenie ping ::1, aby wysłać pakiety ping na adres sprzężenia zwrotnego.
Jeśli wynik działania polecenia ping jest niepomyślny, sprawdź, czy do interfejsu o nazwie Pseudointerfejs sprzężenia zwrotnego przypisano adres ::1.
3. Użyj następującego polecenia, aby wysłać pakiety ping na lokalny dla łącza adres IPv6 komputera:
pingAdres%IDstrefy
gdzie parametr Adres jest adresem lokalnym dla łącza, a parametr IDstrefy jest indeksem interfejsu, do którego przypisano ten adres lokalny dla łącza. Adres lokalny dla łącza zaczyna się od FE80.
Jeśli wynik działania polecenia ping jest niepomyślny, sprawdź poprawność adresu i indeksu interfejsu.
4. Użyj następującego polecenia, aby wysłać pakiety ping na lokalny dla łącza adres innego hosta podłączonego do tego samego łącza (które nazywane jest również podsiecią):
pingAdres%IDstrefy
gdzie parametr Adres jest lokalnym dla łącza adresem innego hosta, a parametr IDstrefy jest indeksem interfejsu, z którego mają zostać wysłane pakiety ping.
Jeśli wynik działania polecenia ping jest niepomyślny, zweryfikuj lokalny dla łącza adres hosta i identyfikator zakresu.

Aby przetestować łączność IPv6 przy użyciu polecenia ping
1. Aby uzyskać konfigurację IPv6 dla komputera, otwórz okno Wiersz polecenia i wpisz polecenie netsh interface ipv6 show interfaceNazwaInterfejsu, gdzieNazwaInterfejsu to nazwa interfejsu na komputerze. Jeśli interfejs nosi na przykład nazwę Połączenie lokalne, należy wpisać:
netsh interface ipv6 show interface "Połączenie lokalne"
Jeśli protokół IPv6 nie jest zainstalowany, zobacz sekcję Tematy pokrewne.
2. Użyj jednego z następujących poleceń, aby wysłać pakiety ping do innego węzła IPv6:
- Aby wysłać pakiety ping na lokalny dla łącza adres innego węzła podłączonego do tego samego łącza (które nazywane jest również podsiecią):
pingAdres%IDstrefy
gdzie parametr Adres jest lokalnym dla łącza adresem innego węzła, a parametr IDstrefy jest zazwyczaj indeksem interfejsu, z którego mają być wysyłane pakiety ping. Indeks interfejsu można uzyskać z danych wyjściowych polecenia netsh interface ipv6 show interface.
Jeśli wynik działania polecenia ping jest niepomyślny, sprawdź poprawność lokalnego dla łącza adresu węzła i identyfikatora strefy.
- Aby wysłać pakiety ping na lokalny dla witryny adres innego węzła:
pingAdres%IDstrefy
gdzie parametr Adres jest lokalnym dla witryny adresem innego węzła, a parametr IDstrefy jest identyfikatorem witryny (identyfikatorem strefy dla witryny) pochodzącym z danych wyjściowych polecenia netsh interface ipv6 show interface level=verbose. Jeśli nie są używane identyfikatory witryn lub jest tylko jedna witryna, fragment polecenia %IDstrefy nie jest wymagany.
Jeśli wynik działania polecenia ping jest niepomyślny, sprawdź poprawność lokalnego dla witryny adresu węzła i identyfikatora strefy.
- Aby wysłać pakiety ping na adres globalny innego węzła:
pingAdres
gdzie parametr Adres jest adresem globalnym innego węzła.
Jeśli wynik działania polecenia ping jest niepomyślny, sprawdź poprawność adresu globalnego węzła.
- Aby wysłać pakiety ping do innego węzła, używając jego nazwy:
ping -6Nazwa
gdzie parametr Nazwa określa nazwę, która może być przetłumaczona na adres IPv6 na podstawie wpisów z lokalnego pliku Hosts lub rekordów zasobów AAAA należących do infrastruktury systemu nazw domen (DNS, Domain Name System). Jeśli do identyfikacji hosta docelowego jest używana nazwa, a nie adres IPv6, konieczne jest dodanie parametru -6, który sprawia, że będą stosowane adresy IPv6.
Jeśli wynik działania polecenia ping jest niepomyślny, sprawdź, czy nazwa może być rozpoznana jako adres IPv6.

Aby wyświetlić konfigurację interfejsu
Otwórz okno wiersza polecenia.
W wierszu polecenia wpisz polecenie netsh, a następnie naciśnij klawisz ENTER.
Wpisz polecenie interface ipv6, a następnie naciśnij klawisz ENTER.
Wpisz show interface [interface=]ciąg_znaków

NETSH - wyświetla informacje o połączeniach sieciowych.

Przykłady zastosowania polecenia netsh:
netsh inteface ipv4>show config
netsh inteface ipv4>show interface
netsh inteface ipv6>show config
netsh => interface => ipv4 => set address "Połączenie lokalne" static 192.168.10.21 255.255.255.0 192.168.10.1
netsh => interface => ipv4 => set address "Local Area Connection" dhcp
netsh => interface => ip => show
netsh interface ip reset

netsh firewall show config - Wyświetla aktualną konfigurację zapory systemu Windows.
netsh firewall reset - Kasuje aktualną konfiguracje zapory wprowadzaną aż od momentu instalacji systemu Windows
